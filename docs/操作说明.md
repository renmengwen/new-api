# 普通用户使用说明

这份说明面向普通用户。如果你只是想把本站当成一个 OpenAI 兼容接口来使用，最短流程就是：

1. 登录账号
2. 创建 API Key（令牌）
3. 填写 Base URL
4. 选择模型
5. 用终端或支持 OpenAI 兼容协议的客户端发起调用

## 一句话配置模板

- `API Key`：你在站内创建并复制的 `sk-xxxx`
- `Base URL`：通常填写 `https://www.linkaihub.com/v1`
- `Model`：从“模型广场”或“操练场”里看到的模型名称

## 第一步：先确认账号可用

登录后，建议先确认这几件事：

- 你的账号分组可以使用目标模型
- 你能在“模型广场”里看到可用模型

如果不确定模型是否能用，先去“操练场”试一次最稳妥。

## 第二步：创建 API Key

进入“控制台”或侧边栏的“令牌管理”，点击“添加令牌”。

建议普通用户这样填写：

- `名称`：按用途命名，比如 `chatbox`、`cli`、`test`
- `令牌分组`：一般保持默认；如果管理员给你开放了 `auto`，也可以选 `auto`
- `跨分组重试`：普通用户不清楚时保持默认即可
- `过期时间`：长期自用可选“永不过期”，临时使用建议设置过期时间
- `无限额度`：如果你想单独限制这个 Key 的最大消耗，就不要勾选
- `模型限制列表`：不懂就留空，留空通常表示不额外限制

创建完成后，在令牌列表里复制完整密钥。实际调用时，密钥格式应当是：

```text
sk-xxxxxxxxxxxxxxxx
```

## 第三步：Base URL 怎么填

这是最容易出错的地方。

大多数 OpenAI 兼容客户端，`Base URL` 应填写：

```text
https://www.linkaihub.com/v1
```

如果你是直接在终端里手写请求地址，那么完整接口通常是：

```text
https://www.linkaihub.com/v1/chat/completions
```

可以这样理解：

- 客户端让你填 `Base URL` / `API Base` / `OPENAI_BASE_URL` 时，通常填 `https://www.linkaihub.com/v1`
- 某些客户端让你填的是“站点地址”“服务器地址”或“Host”，则可能要填 `https://www.linkaihub.com/`
- 最常见错误是把地址写成 `https://www.linkaihub.com/v1/v1`

拿不准时，优先看这个客户端是不是 OpenAI 兼容写法：

- 如果它会自己请求 `/chat/completions`、`/responses` 这类路径，一般就填 `.../v1`
- 如果它要求你手动填写完整接口地址，就直接填完整路径

## 第四步：确认模型名称

调用前，先去“模型广场”或“操练场”确认模型名称。调用请求里的 `model` 必须和站内看到的一致。

例如你在站里看到的是：

```text
gpt-4o-mini
deepseek-chat
claude-3-5-sonnet
gemini-2.5-pro
```

那么请求里的 `model` 就必须写对应名字，不能自己随便改。

## 第五步：用终端调用

### 方式一：`curl`

```bash
BASE_URL="https://www.linkaihub.com"
API_KEY="sk-你的真实密钥"
MODEL="你要使用的模型名"

curl "${BASE_URL}/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ${API_KEY}" \
  -d '{
    "model": "'"${MODEL}"'",
    "messages": [
      {
        "role": "user",
        "content": "你好，简单介绍一下你自己"
      }
    ]
  }'
```

### 方式二：PowerShell

```powershell
$baseUrl = "https://你的站点域名"
$apiKey = "sk-你的真实密钥"
$model = "你要使用的模型名"

$body = @{
  model = $model
  messages = @(
    @{
      role = "user"
      content = "你好，简单介绍一下你自己"
    }
  )
} | ConvertTo-Json -Depth 5

Invoke-RestMethod `
  -Method Post `
  -Uri "$baseUrl/v1/chat/completions" `
  -Headers @{
    Authorization = "Bearer $apiKey"
    "Content-Type" = "application/json"
  } `
  -Body $body
```

## 第六步：给 CLI 工具配置环境变量

很多支持 OpenAI 兼容协议的 CLI 工具，只要改这两个环境变量就能接入本站：

### Bash / zsh

```bash
export OPENAI_API_KEY="sk-你的真实密钥"
export OPENAI_BASE_URL="https://www.linkaihub.com/v1"
```

### PowerShell

```powershell
$env:OPENAI_API_KEY = "sk-你的真实密钥"
$env:OPENAI_BASE_URL = "https://www.linkaihub.com/v1"
```

然后在对应 CLI 工具里选择一个你站内可用的模型即可。

## 常见问题

### 1. 提示 401 / Unauthorized

通常是以下原因：

- API Key 复制错了
- 漏掉了 `sk-` 前缀
- 令牌已被禁用、过期，或额度耗尽

### 2. 提示 404 / 接口不存在

优先检查 Base URL：

- 是否少了 `/v1`
- 是否多写成了 `/v1/v1`
- 是否把完整接口地址和 Base URL 混用了

### 3. 提示模型不存在或无权限

通常说明：

- `model` 名称写错了
- 你的账号分组不能用这个模型
- 你的令牌被设置了模型限制

先去“模型广场”确认模型名，再检查当前令牌配置。

### 4. 提示额度不足

这一般是账户额度或令牌额度不足。去“钱包管理”或“使用日志”里检查即可。

## 推荐的实际使用顺序

如果你是第一次用，建议按这个顺序来：

1. 在“模型广场”确认模型
2. 在“操练场”测试一次
3. 在“令牌管理”创建并复制 API Key
4. 在客户端里填写 `Base URL = https://www.linkaihub.com/v1`
5. 先用 `curl` 跑通，再接入其他客户端或 CLI

这样最不容易出错。
