# 代理商与管理员弹框布局优化设计

## 背景

当前代理商、管理员页面的新增/编辑/查看弹框都把内容拆成了多张上下堆叠的卡片，导致：

- 基础信息和备注信息视觉上像两块独立内容，缺少整体性
- 备注区块宽度和上方表单区域不齐，版式松散
- 查看弹框层级过多，在深色主题下显得碎

## 目标

- 新增/编辑弹框改成单卡分段布局
- 保留“基础信息”“备注信息”等段标题，但不再拆成独立卡片
- 查看弹框保留顶部摘要卡，下方改成单主体卡内分段
- 不改接口、不改字段、不改交互逻辑

## 设计

### 新增/编辑弹框

- 保留现有弹框宽度、标题和底部操作区
- 将原先两张 `sectionStyle` 卡片合并成一个主体容器
- 主体容器内部继续使用两个段落：
  - 基础信息
  - 备注信息
- 备注信息段通过顶部边框与上方字段区分开，并保持整行宽度

### 查看弹框

- 保留顶部摘要卡，用于展示名称、账号和状态
- 将原先多个详情卡合并为一个主体卡
- 主体卡内按信息类型分段：
  - 账号资料
  - 备注信息
  - 额度摘要（仅代理商）
- 段与段之间使用顶部边框分隔，减少卡片层级

## 实施范围

- `web/src/pages/AdminAgentsPageV2/index.jsx`
- `web/src/pages/AdminManagersPageV2/index.jsx`
- 新增一个 source test 锁定布局结构

## 非目标

- 不调整字段校验规则
- 不改变查看弹框的字段集合
- 不改统一 footer 组件
