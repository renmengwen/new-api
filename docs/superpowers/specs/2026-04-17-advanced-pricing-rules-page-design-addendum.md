# 高级定价规则页面设计补充说明

## 1. 目的

本补充说明用于细化《高级定价规则页面设计》中已经确认但尚未展开的页面结构与交互细节，重点补充：

- 页面整体布局选择
- 文本分段规则编辑器结构
- 媒体任务规则编辑器结构
- 价格设置页与高级定价规则页的联动方式

本补充说明与主规格共同生效。

---

## 2. 页面整体布局

高级定价规则页采用：

- 左侧模型列表
- 右侧规则编辑器

即配置中心式布局，而不是详情页式的上下布局。

### 选择该布局的原因

- 适合大量模型频繁切换编辑
- 左侧切模型效率更高
- 右侧能保留足够宽度容纳复杂规则编辑器
- 更符合后台“配置中心”心智

---

## 3. 页面结构细化

### 3.1 左侧模型列表区

用于选择当前编辑模型。

建议字段：

- 模型名称
- 当前计费模式
- 是否存在固定价格配置
- 是否存在高级规则配置
- 规则类型

支持能力：

- 模型名称搜索
- 按计费模式筛选
- 按规则类型筛选

列表中每个模型建议以标签方式展示：

- 当前生效模式：固定按量 / 固定按次 / 高级规则
- 规则类型：文本分段 / 媒体任务

### 3.2 右上模型摘要区

规则编辑区顶部增加“模型摘要卡片”，用于展示当前选中模型的关键状态。

建议显示：

- 当前模型名称
- 当前生效计费模式
- 是否存在固定价格配置
- 是否存在高级规则配置
- 当前规则类型
- 返回价格设置入口
- 切换当前生效模式入口

### 3.3 右下规则编辑区

根据规则类型切换不同编辑器：

- 文本分段规则编辑器
- 媒体任务规则编辑器

不做统一万能表单。

### 3.4 预览区

建议显示：

- 当前规则摘要
- 保存后会写入哪些规则数据
- 当前模型的规则摘要
- 使用日志与任务日志的说明预览

---

## 4. 文本分段规则编辑器

文本分段规则编辑器用于表达 Gemini 与火山文本模型等分段文本规则。

结构建议：

- 顶部规则摘要
- 中间规则列表
- 新增/编辑规则抽屉
- 命中预览面板

### 4.1 顶部规则摘要

展示：

- 规则类型：文本分段规则
- 当前已配置规则数
- 是否存在默认兜底规则
- 命中顺序说明

### 4.2 规则列表

每条规则一行，建议字段：

- 优先级
- 条件摘要
- 输入价格
- 输出价格
- 缓存读取价格
- 缓存创建价格
- 服务层级
- 状态
- 操作

其中“条件摘要”只展示规则概要，例如：

- 输入长度 `[0,32]`
- 输入长度 `(32,128]`
- 输入长度 `[0,32] + 输出长度 [0,0.2]`

### 4.3 新增/编辑规则抽屉

抽屉内分为两部分：

- 条件区
- 价格区

条件区支持：

- 输入长度最小值
- 输入长度最大值
- 输出长度最小值
- 输出长度最大值
- 服务层级
- 缓存相关条件

价格区支持：

- 输入价格
- 输出价格
- 缓存读取价格
- 缓存创建价格

页面表达为价格，内部仍由后端转换为系统实际使用结构。

### 4.4 规则校验

至少需要校验：

- 区间不能明显重叠
- 最小值不能大于最大值
- 至少包含一个条件
- 至少填写输入价格
- 若填写输出/缓存价，必须有基础输入价
- 优先级不能冲突

### 4.5 命中预览

抽屉底部增加“命中预览”面板，允许输入：

- 输入 token 数
- 输出 token 数
- 服务层级

系统应即时预览：

- 命中哪条规则
- 将使用的输入/输出价格
- 预计生成的日志说明摘要

## 5. 媒体任务规则编辑器

媒体任务规则编辑器用于表达火山视频模型等任务型复杂规则。

其结构与文本规则类似，但规则维度不同：

- 顶部规则类型说明
- 规则列表
- 新增/编辑规则抽屉
- 任务计费预览

### 5.1 顶部规则类型说明

需明确显示：

- 该模型为媒体任务规则
- 最终按任务 usage / total_tokens 结算
- 使用日志与任务日志将基于命中规则生成说明

### 5.2 规则列表

建议字段：

- 优先级
- 推理模式：在线 / 离线
- 输入视频：包含 / 不包含
- 音频：有声 / 无声
- 分辨率
- 宽高比
- 输出时长
- Draft 折扣
- 单价
- 最低 token 阈值
- 状态
- 操作

### 5.3 新增/编辑规则抽屉

抽屉内条件区建议支持：

- 推理模式
- 是否包含输入视频
- 是否包含音频
- 分辨率
- 宽高比
- 输出时长区间
- 输入视频时长区间（如启用）
- 是否 Draft
- 是否启用最低 token 阈值

价格区建议支持：

- 单价
- 最低 token 阈值
- Draft 折算系数
- 备注说明

这里不允许运营手写公式，仅配置命中条件与参数，公式逻辑由系统内置。

### 5.4 任务计费预览

应提供预览输入项：

- 在线 / 离线
- 是否输入视频
- 是否音频
- 分辨率
- 宽高比
- 输出时长
- Draft 与否
- usage.total_tokens

系统应预览：

- 命中哪条规则
- 单价是多少
- 是否命中最低 token 阈值
- 最终按哪个 token 值结算
- 使用日志将如何展示

## 6. 价格设置页与高级定价规则页联动

价格设置页仍是运营主入口。

建议在价格设置页列表新增两列：

- `计费模式`
- `规则类型`

其中：

- `计费模式` 取值为：固定按量 / 固定按次 / 高级规则
- `规则类型` 仅在高级规则模式下显示：文本分段 / 媒体任务

### 6.1 操作列建议

建议增加两个操作入口：

- 编辑固定价格
- 编辑高级规则

并根据当前模式高亮主入口。

### 6.2 模式切换建议

模式切换应当放在价格设置页列表或右侧详情顶部，不建议藏到高级规则页里。

切换时规则：

- 不删除另一套配置
- 仅切换当前生效模式
- 切换前弹出确认提示

确认提示应明确：

- 切换后新请求将按新模式结算
- 历史日志与账单不会回刷

### 6.3 回滚方式

高级规则上线后如果需要回滚：

- 直接将模型计费模式切回固定按量或固定按次
- 另一套配置应已保留，无需重新录入

这也是“配置保留、单模式生效”设计的核心价值。

### 6.4 状态展示

为减少误操作，价格设置页建议展示如下状态文案：

- 当前按固定按量生效
- 当前按固定按次生效
- 当前按高级规则生效
- 已存在高级规则配置（未生效）
- 已存在固定价格配置（未生效）

同时在高级规则页顶部也需展示：

- 当前模型当前生效模式
- 若高级规则已配置但未生效，需提示：
  - 已配置高级规则，但当前未生效
  - 切换到高级规则后，新请求才会按规则结算

---

## 7. 页面交互路径

推荐标准操作路径：

1. 在 `价格设置` 页查看模型当前 `计费模式`
2. 点击“编辑高级规则”进入高级定价规则页
3. 在左侧模型列表中切换目标模型
4. 在右侧规则编辑器中修改规则
5. 在预览区确认命中摘要与日志说明
6. 保存规则
7. 返回价格设置页切换或确认当前生效模式

这样可以保证：

- 主入口统一
- 专业编辑分流
- 生效模式切换清晰
