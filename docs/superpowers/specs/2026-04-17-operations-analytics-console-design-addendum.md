# 运营分析台设计补充说明
日期：2026-04-17

## 1. 适用范围

本补充说明用于覆盖并细化主规格 [2026-04-17-operations-analytics-console-design.md](/D:/code3/new-api/docs/superpowers/specs/2026-04-17-operations-analytics-console-design.md:1) 中以下部分：

- 顶部汇总卡片
- 按模型 Tab 的图表与 token 展示语义
- 按用户 Tab 的图表指标与布局
- 按日 Tab 的费用图显示规则
- 本轮新增的测试与验收点

如本补充说明与主规格存在冲突，以本补充说明为准。

## 2. 顶部汇总卡片补充

### 2.1 卡片数量

顶部汇总卡片从 4 张调整为 5 张，顺序固定为：

1. 总调用量
2. 总费用
3. 活跃用户
4. 活跃模型
5. 总 Token 量

### 2.2 视觉方向

顶部汇总卡片采用“方案 A”：

- 每张卡片使用更强层级的运营分析视觉风格，而不是当前偏平的浅底卡片
- 每张卡片引入 Semi icon，图标由前端按语义选择合适组件
- 允许使用彩色渐变底或高饱和浅底，确保 5 张卡片之间的视觉区分度
- 数值作为主视觉，标题、图标、辅助文案、同比信息形成清晰层级

本次设计不锁定具体图标组件名，但要求：

- 总调用量、总费用、活跃用户、活跃模型、总 Token 量五张卡片必须各自具备独立图标
- 颜色、图标、标题、数值之间具备清晰映射关系，避免再次出现“整页配色太单调”的问题

### 2.3 总 Token 量口径

新增卡片 `总 Token 量`，统计口径如下：

- `total_tokens = sum(logs.prompt_tokens + logs.completion_tokens)`

该字段必须由后端 summary 聚合接口返回，不允许由前端临时拼装。

### 2.4 总 Token 量的自然周同比

`总 Token 量` 与前面四张卡片保持一致：

- `近7天 / 默认视图`：显示自然周同比
- `今日`：不显示自然周同比
- `自定义`：不显示自然周同比

同比口径与主规格一致：

- 卡片主值始终跟随当前筛选时间范围
- 同比只在 `近7天` 视图展示
- `wow.total_tokens` 由后端汇总接口返回，不由前端推导

## 3. 按模型 Tab 补充

### 3.1 模型调用量图的轴向修正

当前“模型调用量柱状图”使用横向柱图，导致 X 轴显示的是调用次数而不是模型名称。

调整后：

- 图表改为竖向柱状图
- X 轴显示模型名称
- Y 轴显示调用次数

这样可以消除“轴向显示错位”的理解成本。

### 3.2 视觉 / 任务模型的 Token 显示语义

部分视觉模型、视频模型、任务型模型当前日志中可能出现：

- `logs.quota > 0`
- `logs.prompt_tokens = 0`
- `logs.completion_tokens = 0`

这类场景不应在表格中直接显示普通数值 `0`，否则运营会误以为系统拿到了 token 明细，只是数值恰好为零。

按模型表格中的 `输入 Token / 输出 Token` 显示规则补充为：

- 普通文本模型：显示真实数值
- 若 `prompt_tokens = 0 && completion_tokens = 0 && total_cost > 0`：显示 `暂无 token 数据`
- 若真实值就是 `0` 且不存在费用口径冲突：继续显示 `0`

设计约束：

- 不允许通过费用反推 token
- 不允许做估算换算
- 不允许伪造 token 数值

页面要表达的是“当前日志源没有 token 明细”，而不是“token 为 0”。

## 4. 按用户 Tab 补充

### 4.1 图表指标调整

按用户 Tab 的两个图表从：

- Top 用户费用
- Top 用户调用量

调整为：

- Top 用户使用金额
- Top 用户 Token 总量

其中：

- `用户 Token 总量 = sum(logs.prompt_tokens + logs.completion_tokens)`

### 4.2 图表轴向修正

两个用户图表均调整为竖向柱状图：

- X 轴显示用户名称
- Y 轴显示金额 / Token 总量

用户名称显示规则：

- 优先显示 `username`
- 若用户名为空，则显示 `#user_id`

### 4.3 用户表格铺满整行

按用户 Tab 布局调整为两段式：

- 第一行：两个图表并排展示
- 第二行：用户表格单独占满整行宽度

不再允许用户表格与上方图表共用半宽网格，避免当前“表格只显示一半宽度”的问题。

### 4.4 后端聚合字段补充

为了支撑“Top 用户 Token 总量”图表，按用户聚合接口需要新增返回字段：

- `total_tokens`

统计口径：

- `sum(logs.prompt_tokens + logs.completion_tokens)`

## 5. 按日 Tab 补充

按日 Tab 继续保持“无表格，只显示趋势图”的定位。

其中“按日费用”折线图增加以下明确规则：

- hover tooltip 只显示金额
- 不显示 token 信息
- Y 轴刻度标签也必须按金额格式化显示

也就是说：

- 费用图的 tooltip formatter 使用金额格式
- 费用图的 Y 轴 label formatter 也使用金额格式
- 不允许沿用通用数字格式，避免出现“费用图却显示为裸数值”的问题

其他三张按日图：

- 按日调用次数
- 按日活跃用户
- 按日活跃模型

继续按整数/次数口径展示，不做金额格式化。

## 6. 测试与验收补充

除主规格中的测试要求外，本轮新增以下验收点：

1. 顶部卡片数量为 5 张，包含 `总 Token 量`
2. `总 Token 量` 在 `近7天` 下显示自然周同比，在 `今日 / 自定义` 下不显示同比
3. 按模型表格中，视觉 / 任务模型在“有费用但无 token 明细”时显示 `暂无 token 数据`
4. 按模型调用量图改为模型名称位于 X 轴、调用次数位于 Y 轴
5. 按用户图表改为“使用金额 + Token 总量”
6. 按用户两个图表的 X 轴显示用户名称，不再显示数值轴标签代替名称
7. 按用户表格单独占满整行宽度
8. 按日费用图的 hover 与 Y 轴刻度都按金额显示，不显示 token

## 7. 范围说明

本补充说明不改变主规格中已确认的以下前提：

- 页面名称仍为 `运营分析台`
- 代理商范围仍为“归属自己名下 token/用户”的调用
- 平均响应时间仍取使用日志中的 `use_time`
- 自然周同比规则仍为固定自然周同比
- 顶部卡片主值仍跟随当前筛选时间范围
