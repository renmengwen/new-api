# 运营模型定价配置参考：火山方舟与 Gemini

本文面向运营人员，目标是给出当前系统里 **火山方舟** 与 **Gemini** 模型的推荐定价配置方式。

适用页面：

- `系统设置 -> 分组与模型定价设置 -> 价格设置`
- `系统设置 -> 分组与模型定价设置 -> 高级定价规则`

## 1. 建议

### 1.1 火山方舟

当前系统可以较好覆盖：

- 大语言模型 -> 在线推理（常规 token）
- 视频生成模型

建议：

- 火山文本分段模型：使用 `高级定价规则 -> 文本分段规则`
- 火山视频模型：使用 `高级定价规则 -> 媒体任务规则`
- 火山少量非分段文本模型：继续用 `价格设置` 页的固定价格

### 1.2 Gemini

当前系统可以较好覆盖：

- Gemini Pro / 2.5 Pro 这类按输入 token 长度分层的模型
- Gemini Flash / Flash-Lite 里“按输入模态不同单价”的文本口径

当前系统**不能精确覆盖**的部分：

- `gemini-3.1-flash-live-preview` 的实时音频/视频按秒费用
- `gemini-3.1-flash-image-preview`、`gemini-3-pro-image-preview` 的图片输出档位费用
- 所有 Gemini 的 `Google Search / Maps grounding` 附加费用

这些模型本文仍给出“简化示例”，但会明确标注边界。

---

## 2. 使用前必须知道的规则

### 2.1 火山文本价格表里的区间单位是“千 token”

火山文档里常见写法：

- `输入长度 [0,32]`
- `输入长度 (32,128]`
- `输出长度 [0,0.2]`

在当前系统里，保存时要写成**实际 token 数**：

- `输入长度 [0,32]` -> `输入最小值=0`, `输入最大值=32000`
- `输入长度 (32,128]` -> `输入最小值=32001`, `输入最大值=128000`
- `输出长度 [0,0.2]` -> `输出最小值=0`, `输出最大值=200`

### 2.2 火山文档里的价格如果是人民币，要先按你们当前计费汇率换算

如果你们系统内部仍然统一按 USD 维护价格，则：

- 火山文档里的原始单价是 **人民币**
- 真正填到系统里的值，应先除以你们当前生效汇率

例如：

- 如果当前 `1 USD = 7.30 CNY`
- 文档单价是 `46`
- 那么系统里应填 `46 / 7.30 = 6.301369`

本文以下在火山部分默认先写**官方原始单价**，你们落配置时再按当前汇率换算。

### 2.3 Gemini 本文默认按“标准档”给示例

Gemini 文档里还有：

- 批量
- Flex
- 优先级

本文默认只给“标准档”配置示例。  
如果后续要支持非标准档，请在同一模型下额外增加规则，并用 `服务层级` 区分。

### 2.4 Grounding 当前暂不建议配置

当前系统里 `per_1000_calls` 规则还不能稳定作为“附加费”叠加在 token 计费之上，因此：

- `Google Search grounding`
- `Google Maps grounding`

统一建议：

- **暂不支持，不建议填入高级定价规则**

## 4. 火山方舟：视频模型

### 4.1 通用建议

统一使用：

- 规则类型：`媒体任务`
- 任务类型：`video_generation`
- 计费单位：`每100万个 tokens`（`per_million_tokens`）

### 4.2 `doubao-seedance-2.0`

建议先配置最常用的 4 条规则：

#### 规则 1：无输入视频，480p，16:9，输出 5 秒

- `输入视频=no`
- `分辨率=480p`
- `宽高比=16:9`
- `输出时长最小值=5`
- `输出时长最大值=5`
- `单价=46`

#### 规则 2：无输入视频，720p，16:9，输出 5 秒

- `输入视频=no`
- `分辨率=720p`
- `宽高比=16:9`
- `输出时长最小值=5`
- `输出时长最大值=5`
- `单价=46`

#### 规则 3：输入含视频，480p，16:9，输出 5 秒，输入视频 2~15 秒

- `输入视频=yes`
- `分辨率=480p`
- `宽高比=16:9`
- `输出时长最小值=5`
- `输出时长最大值=5`
- `输入视频时长最小值=2`
- `输入视频时长最大值=15`
